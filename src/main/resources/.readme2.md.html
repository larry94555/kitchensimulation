search-redirects-services

Search Redirect Services is the backend service for the new Redirects feature of the CXFind tool. This tool is used by Category Specialists and Product Operations to improve the search experience through targeted redirects.

Details can be found here: https://confluence.walmart.com/display/LABSPOLSEA/Redirects+Design+and+Architecture

Build Status

Requirements

For building and running the application you need:

  • JDK 1.8
  • Maven 3

Running locally from Command Line

  1. Clone the Repo.
  2. Build running mvn clean install
  3. To run, copy the following to the command line, make changes as needed for the path, and then hit ENTER:

java -Dcom.walmart.platform.config.runOnEnv=dev -Dcom.walmart.platform.config.appName="search-redirects-services" -Dapp.name="search-redirects-services" -Xdebug -Druntime.context.environmentType=dev -Druntime.context.appName=search-redirects-services -Druntime.context.system.property.override.enabled=true -jar ~/GitHub/search-redirects-services/target/search-redirects-services-0.0.1-SNAPSHOT.jar

  1. You can verify it is running by opening a browser to the following url:

http://localhost:8080/actuator/health

If it is running, you should see this:

{

"status": "UP"

}

Setup for Eclipse

  1. Clone the Repo.
  2. Import search-redirects-services as an existing maven project

Within Eclipse, select File-> Import..., Existing Maven Projects, Next >, go the top of the project ./search-redirect-services/.., select search-redirect-services, click Open (which should have turned blue).

  1. To build:

In Project Explorer, right click search-redirects-services, select Run As, Maven Build... Set for Goals: clean install and then hit Run.

  1. To run:

In Project Explorer, right click search-redirects-services, select Run As, Run Configurations... Click (x)= Arguments tab, Add the following to VM arguments (entering userid and making changes to path):

-Dcom.walmart.platform.config.runOnEnv=dev -Dcom.walmart.platform.config.appName="search-redirects-services" -Dapp.name="search-redirects-services" -Xdebug -Druntime.context.environmentType=dev -Druntime.context.appName=search-redirects-services -Druntime.context.system.property.override.enabled=true -jar /Users//GitHub/redirect-tests/search-redirects-services/target/search-redirects-services-0.0.1-SNAPSHOT.jar

Click Apply

  1. You can verify it is running by opening a browser to the following url:

http://localhost:8080/actuator/health

If it is running, you should see this:

{

"status": "UP"

}
